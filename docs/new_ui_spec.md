## 0. 改善の基本方針

| 軸 | 方針 |
|----|------|
| **初見の導線** | アプリを開いた瞬間から「曲を追加する」「再生する」の2アクションに集中させる |
| **操作の直感性** | 説明なしで意味が伝わるアイコン・ラベル・状態表示に統一する |
| **一貫性** | アプリ全体でボタン配置・配色・タイポグラフィのルールを定める |

### 1-C. ボタン・アクション配置ルール ✅ 新画面で適用済み

```
ナビゲーションバー：
  左（Back / Cancel） — 「戻る」「キャンセル」系
  右（Primary Action） — 最重要1アクションのみ（追加・完了・保存）

スワイプアクション：
  「削除」のみ。赤背景（.destructive）

編集モード：
  「並び替え」専用。削除はスワイプに一本化。
```

### 1-D. 状態表示ルール ✅ 再生画面で適用済み

```
1. アイコン変化  — SF Symbols で状態を表現
2. アクセントカラー — ON: AppColor.accent / OFF: AppColor.textSecondary
3. ラベル補足  — シャッフル/リピートはテキストラベルも表示
```

## 6. 技術的改善方針（実装メモ）

### 状態管理
- グローバル変数（`SHUFFLE_FLG`, `repeatState` など）を将来的に `MusicPlayerState` 構造体に集約
- 現状は `NowPlayingMusicLibraryData` / `displayMusicLibraryData` の2つのグローバル構造体で管理

### 広告
- 各 ViewController が個別にロードしている広告の `AdManager` シングルトン化は未着手
- 現状: 各画面で `GADBannerView` などを直接保持

### キーボード管理 ✅ 新画面で統一パターン適用済み
```swift
// NotificationCenter パターン（keyboardLayoutGuide より信頼性が高い）
NotificationCenter.default.addObserver(self, selector: #selector(keyboardWillChange(_:)),
    name: UIResponder.keyboardWillChangeFrameNotification, object: nil)
// scrollView.contentInset.bottom = kbHeight で対応
```
旧画面は `keyboardLayoutGuide` を使っているものが残存。順次移行。

### カラーテーマ
- 現状: `NAVIGATION_COLOR[9][5]` の2次元配列 + `COLOR_THEMA` enum
- 目標: `AppTheme.current.accent` などのプロパティアクセスに統一（未着手）
